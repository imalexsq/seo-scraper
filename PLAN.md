## What this project is

A standalone Python scraper. It has one job: collect competitive intelligence and write it
to a SQLite database. It does not generate SEO titles, does not talk to Shopify, and does not
output CSVs. That's all handled by the `seo-optimizer` Cowork skill, which reads from this
project's database.

```
seo-scraper (this project)          seo-optimizer (Cowork skill)
─────────────────────────           ────────────────────────────
Google SERP  ─┐                     GA4 + Shopify pull
              ├─→ SQLite DB  ──────→ load competitor context
Product pages ─┘                    Claude title generation
                                    CSV review + Shopify write
```

When this project has not been run, the skill operates in Claude-only mode (no competitor
context). When it has been run, the skill loads the DB and injects competitor intel into every
title generation prompt. The skill never writes to this DB.

## Phases

### Phase 2 — SERP + product page scraping (build next)

Scrapes Google for each product's keywords, follows every result URL, extracts and classifies
competitor metal claims. Outputs structured data to SQLite.

**Inputs:** keyword list (generated by seo-optimizer skill, stored in `data/keywords.json`)
**Outputs:** `data/scraper.db` — SERP snapshots, raw page data, classified material per URL

### Phase 3 — Competitor monitoring (scheduled, ongoing)

Extends Phase 2 into a persistent intelligence layer. Runs weekly on a cron schedule.
Diffs each run against the previous one, detects changes, sends a Telegram digest.

**Inputs:** `config/competitors.json` — tracked domains and keywords
**Outputs:** delta records in `data/scraper.db` + weekly Telegram message via Lumi/OpenClaw

## OSS Stack

Do not build custom Playwright infrastructure. Use these instead:

### Crawlee Python — crawling engine
**Repo:** https://github.com/apify/crawlee-python
**Replaces:** custom `google_serp.py`, `page_scraper.py`, `proxy_manager.py`, `stealth.py`

Crawlee handles proxy rotation, request queuing, retries, rate limiting, and anti-bot
fingerprinting out of the box. We write crawlers on top of it, not plumbing underneath it.

```bash
pip install crawlee[playwright]
playwright install chromium
```

```python
# Example: SERP crawler using Crawlee
from crawlee.playwright_crawler import PlaywrightCrawler, PlaywrightCrawlingContext

crawler = PlaywrightCrawler(
    proxy_configuration=ProxyConfiguration(proxy_urls=[
        "http://user:pass@brd.superproxy.io:33335"  # Bright Data rotating residential
    ]),
    max_request_retries=3,
    request_handler_timeout=timedelta(seconds=30),
)

@crawler.router.default_handler
async def handler(context: PlaywrightCrawlingContext):
    await context.page.wait_for_load_state("domcontentloaded")
    # extract SERP results or product page data
    ...
```

### ScrapeGraphAI — intelligent data extraction
**Repo:** https://github.com/ScrapeGraphAI/Scrapegraph-ai
**Replaces:** custom CSS selectors and per-site extraction logic

Instead of writing a different parser for Missoma, Monica Vinader, Angara, etc., describe the
data you want in a prompt. ScrapeGraphAI uses Claude Haiku to extract it from any page
structure. Robust to site redesigns — no maintenance.

```bash
pip install scrapegraphai
```

```python
from scrapegraphai.graphs import SmartScraperGraph

graph = SmartScraperGraph(
    prompt="""Extract from this jewelry product page:
    - product_title (string)
    - metal_type: one of solid_gold, vermeil, gold_plated, gold_filled, silver, unclear
    - karat: e.g. 18k, 14k, 9k, or null
    - base_metal: sterling_silver, brass, base_metal, or null
    - price_usd: number or null
    - evidence: the exact phrase from the page that told you the metal type""",
    source=url,
    config={
        "llm": {
            "provider": "anthropic",
            "model": "claude-haiku-4-5-20251001",
            "api_key": ANTHROPIC_KEY,
        },
        "headless": True,
    }
)
result = graph.run()
# { "product_title": "...", "metal_type": "vermeil", "karat": "18k",
#   "base_metal": "sterling_silver", "price_usd": 285.0, "evidence": "18k gold vermeil" }
```

Then wrap in a thin normalisation layer to coerce to our schema and handle extraction failures.

### ~~Scrapoxy~~ — dead, do not use
Scrapoxy announced end of life in 2025. Docker images removed, no maintainer.
Use a commercial proxy provider's rotating endpoint instead. One URL, they handle rotation
server-side. No proxy manager software needed on our side.

## Proxy Setup

Two pools. Both use rotating endpoints — no IP management on our side.

| Pool | Provider | Use | Cost |
|---|---|---|---|
| `serp` | Bright Data residential | Google SERP scraping | ~$15/GB |
| `pages` | IPRoyal residential | Product page scraping | ~$7/GB |

Bright Data is worth the premium for Google specifically — highest CAPTCHA resistance.
IPRoyal is fine for product pages (Shopify stores, mostly no Cloudflare).

```yaml
# config/config.yml
proxies:
  serp:
    endpoint: "http://{user}:{pass}@brd.superproxy.io:33335"
  pages:
    endpoint: "http://{user}:{pass}@geo.iproyal.com:12321"
```

Expected cost: ~$10–30/month running weekly monitoring + monthly SEO refreshes.

If CAPTCHA rate climbs above 15% even on residential: add 2captcha/CapSolver as a fallback.
At our query volume this should not be necessary.

## SQLite Schema

```sql
-- SERP snapshots
CREATE TABLE serp_results (
    id          INTEGER PRIMARY KEY,
    keyword     TEXT NOT NULL,
    position    INTEGER,
    url         TEXT NOT NULL,
    title       TEXT,
    snippet     TEXT,
    scraped_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Raw product page data
CREATE TABLE product_pages (
    id          INTEGER PRIMARY KEY,
    url         TEXT UNIQUE NOT NULL,
    domain      TEXT,
    page_title  TEXT,
    raw_html    TEXT,         -- trimmed to first 10k chars
    scraped_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Classified material per page (ScrapeGraphAI output)
CREATE TABLE classifications (
    id          INTEGER PRIMARY KEY,
    url         TEXT NOT NULL,
    metal_type  TEXT,         -- solid_gold | vermeil | gold_plated | gold_filled | silver | unclear
    karat       TEXT,         -- 18k | 14k | 9k | null
    base_metal  TEXT,         -- sterling_silver | brass | null
    price_usd   REAL,
    evidence    TEXT,         -- exact quote used to classify
    model       TEXT,         -- which model classified it
    classified_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (url) REFERENCES product_pages(url)
);

-- Phase 3: delta tracking
CREATE TABLE deltas (
    id           INTEGER PRIMARY KEY,
    url          TEXT NOT NULL,
    delta_type   TEXT,        -- price_change | metal_claim_change | title_change | new_competitor
    old_value    TEXT,
    new_value    TEXT,
    detected_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    alerted      BOOLEAN DEFAULT FALSE
);
```

## How seo-optimizer reads from this DB

The skill checks for the DB at startup:

```python
import sqlite3, os

DB_PATH = os.path.expanduser("~/seo-scraper/data/scraper.db")

def get_competitor_context(keywords: list[str]) -> str:
    if not os.path.exists(DB_PATH):
        return ""   # skill runs in Claude-only mode
    
    conn = sqlite3.connect(DB_PATH)
    rows = conn.execute("""
        SELECT c.metal_type, c.karat, c.base_metal, c.price_usd,
               c.evidence, p.page_title, s.keyword, s.position
        FROM classifications c
        JOIN product_pages p ON c.url = p.url
        JOIN serp_results s  ON p.url = s.url
        WHERE s.keyword IN ({})
        ORDER BY s.position ASC
        LIMIT 15
    """.format(",".join("?" * len(keywords))), keywords).fetchall()
    conn.close()
    
    if not rows:
        return ""
```

## Key Decisions

| Decision | Default | Revisit when |
|---|---|---|
| Crawlee vs raw Playwright | Crawlee — handles infra we'd write anyway | Never — Crawlee is strictly better |
| ScrapeGraphAI vs CSS selectors | ScrapeGraphAI — no per-site maintenance | If Haiku extraction accuracy drops below 85% |
| Run on Jetson vs desktop | Jetson first — already running, SSH accessible | Playwright too slow on ARM (Nano is old) |
| CAPTCHA solver (2captcha) | Not yet — residential proxies + delays is enough | CAPTCHA rate > 15% on Bright Data |
| Crawlee Cloud (self-hosted) | Not yet — single machine is fine | Volume exceeds what one Jetson/PC can handle |
| Monitoring frequency | Weekly | Daily if a competitor is actively moving on key terms |
